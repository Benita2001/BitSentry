The Problem

Trading Agents Are Flying Blind

Most trading agents connect directly to exchanges with no safety layer between intention and execution.

⚠️

Black Box Agents

When your agent loses money, you don't know why. No decision trail. No accountability. Just a smaller balance.

💥

No Risk Controls

Agents trade without leverage caps, position limits, or loss circuit breakers. One bad trade wipes the account.

📉

Dead Strategies

Strategies degrade silently. Your agent keeps running a broken strategy while you watch the losses mount.

The Solution

5 Layers of Protection

Every trade passes through all 5 layers before touching Bitget.

1

Risk Guardian

5-layer pre-trade check. Leverage cap, position size limit, daily loss circuit breaker, consecutive loss throttle, and market condition filter with live Fear & Greed and volatility data.

PROCEED / REDUCE_SIZE / WAIT / BLOCKED
2

Audit Engine

Every decision logged to SQLite with SHA-256 integrity verification. Tamper-proof. Reproducible. Judges and auditors can verify the full decision trail independently.

integrity_hash: SHA-256
3

Position Monitor

Real-time safety ratings for every open Bitget position based on margin distance and unrealized PnL percentage. Blocks new entries when existing positions are already stressed.

GREEN / YELLOW / RED
4

Strategy Evaluator

Rolling win rate over 7d and 30d windows with profit factor scoring. Detects when strategies degrade before they destroy accounts. Stops the agent before it's too late.

PERFORMING / DEGRADING / DEAD
5

Report Generator

Daily, weekly, and monthly trading summaries sent automatically to Telegram. Your agent reports to you, not the other way around. Includes SHA-256 verified audit integrity status.

Telegram: Daily 23:00 UTC
Market Intelligence

Market-Aware Risk

BitSentry reads live market conditions before approving every trade.

Fear & Greed Index

When Fear & Greed hits Extreme Greed (>75), BitSentry reduces long position sizes by 50%. When Extreme Fear (<25), short sizes are reduced to protect against counter-trend capitulation.

Live index: Loading…

Funding Rate

High positive funding means longs are paying heavily. BitSentry flags this as a warning so agents avoid expensive carry costs and consider timing their entry more carefully.

Source: Bitget futures API via bgc

Volatility Filter

When 24h volatility exceeds 4%, BitSentry caps leverage recommendations to 5x to protect against liquidation in fast-moving and choppy market conditions.

Calculated from: live Bitget OHLCV data
Installation

Get Started in 4 Steps

BitSentry works alongside Bitget Agent Hub with zero conflict.

1

Install Bitget Agent Hub

bash 
npx bitget-hub upgrade-all --target claude
2

Set credentials (same as Agent Hub)

bash 
export BITGET_API_KEY="your_api_key"
export BITGET_SECRET_KEY="your_secret_key"
export BITGET_PASSPHRASE="your_passphrase"
3

Install BitSentry

bash 
pip install bitsentry
4

Add to Claude Code MCP config

json 
{
  "mcpServers": {
    "bitget": {
      "command": "npx",
      "args": ["-y", "bitget-mcp-server"]
    },
    "bitsentry": {
      "command": "npx",
      "args": ["-y", "@0xbeni/bitsentry-mcp"]
    }
  }
}
